# Как вайбинженер ведёт персистентную память

Claude Code умеет хранить память между сессиями (каталог `memory` в `~/.claude/projects/...` или `# заметка` для быстрой записи в CLAUDE.md). Память — мощный инструмент и типичный источник тихого рассинхрона. Правила:

## Что писать в память

- **Незапоминаемые детали**: IP-адреса стендов, имена окружений, порты, URL внутренних дашбордов, «прод-база называется не так, как стейджинг».
- **Договорённости с людьми**: «команда решила не трогать модуль X до Q3», «ревью у нас через PR, не пуш в main».
- **Твои предпочтения как пользователя**: стиль ответов, любимые инструменты, «не предлагать мне Docker, у нас его нет».
- **Выводы дорогих исследований**: если агент три часа разбирался, почему падает сборка на Windows, — вывод должен пережить сессию.

## Что НЕ писать в память

- То, что и так лежит в репозитории (структура кода, команды сборки, конвенции) — этому место в CLAUDE.md проекта, где оно версионируется вместе с кодом. Память, дублирующая репо, устаревает молча и начинает врать.
- То, что важно только для текущей задачи, — умрёт вместе с сессией, и это правильно.
- Секреты. Никогда: ключи, пароли, токены.

## Гигиена

- Одна запись = один факт. Относительные даты («на прошлой неделе») конвертируй в абсолютные.
- Раз в месяц-два проси агента провести ревизию памяти: слить дубли, удалить устаревшее (в комплекте Claude Code есть скилл consolidate-memory).
- Нашёл в памяти ложь — удаляй сразу, не «потом».

## Правило большого пальца

> CLAUDE.md — знания **проекта** (едут с репо, видны команде).
> Память — знания **человека и окружения** (твои, локальные, между проектами).
