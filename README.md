# Комплект вайбинженера

Переносимый набор для работы с Claude Code (Fable 5) по инженерному процессу: скиллы, сабагенты, конфиги, гайд. Что такое «вайбинженер» и как по этому работать — в [GUIDE.md](GUIDE.md).

## Установка (3 шага)

1. **Скопируй шаблон в свой проект:**

   ```powershell
   Copy-Item -Recurse -Force .\project-template\* C:\путь\к\твоему\проекту\
   ```

   (файлы `.mcp.json` и каталог `.claude` — скрытые по имени, но `*` в PowerShell их берёт; проверь, что они скопировались)

2. **Заполни `CLAUDE.md`** — минимум секцию «Стек и команды» (build/test/lint). Без неё агент будет угадывать, как запускать твои тесты.

3. **(Опционально) сделай скиллы и агентов глобальными** — скопируй `project-template\.claude\skills\*` в `~\.claude\skills\` и `project-template\.claude\agents\*` в `~\.claude\agents\`, чтобы они работали во всех проектах.

Затем открой проект в Claude Code — скиллы `/spec`, `/ship`, `/postmortem`, `/vibe-check` появятся автоматически.

## Что лежит и зачем

| Путь | Что это | Зачем |
|---|---|---|
| `GUIDE.md` | Манифест + процесс | Ядро сета: петля Spec→Plan→Build→Verify→Review→Ship→Learn |
| `project-template/CLAUDE.md` | Шаблон правил проекта | Постоянный контекст агента: стек, конвенции, выученные правила |
| `project-template/.mcp.json` | MCP-конфиг | Context7 — актуальные доки библиотек, без ключей |
| `project-template/.claude/settings.json` | Права и хуки | Меньше permission-промптов на безопасное, запрет разрушительного |
| `.claude/skills/spec/` | `/spec` | Хотелка → спецификация с критериями приёмки (агент сперва задаёт вопросы) |
| `.claude/skills/ship/` | `/ship` | Конвейер сдачи: тесты → линт → чтение дифа → анти-reward-hacking → коммит |
| `.claude/skills/postmortem/` | `/postmortem` | Баг → 5 «почему» → правило в CLAUDE.md, чтобы не повторился |
| `.claude/skills/vibe-check/` | `/vibe-check` | Аудит репо на признаки вайбкода (мёртвый код, TODO-кладбища, хардкод) |
| `.claude/agents/scout.md` | Сабагент-разведчик | Read-only, дешёвая модель — карта кода перед планированием |
| `.claude/agents/critic.md` | Сабагент-критик | Свежий контекст, адверсариальное ревью: ломает, а не хвалит |
| `.claude/agents/test-smith.md` | Сабагент-тестописец | Тесты по спецификации, запрет ослаблять существующие тесты |
| `global/memory-seed.md` | Заметка про память | Как вести персистентную память, чтобы она помогала, а не врала |

## Опциональные MCP (добавь в `.mcp.json` под задачу)

- **GitHub** — PR/issues/CI из сессии (или просто установи `gh` CLI);
- **Playwright / Claude in Chrome** — визуальная верификация UI для веба;
- **Hugging Face** — если данные/ML;
- **Sentry, Datadog** — прод-отладка.

Принцип: каждый MCP ест контекст. Не пригодился за неделю — отключи.

## Настройка хуков автоформатирования

В `settings.json` есть закомментированная (через ключ `_disabled_hooks`) заготовка PostToolUse-хука: автоформат файла после каждого Edit/Write. Переименуй `_disabled_hooks` в `hooks` и подставь свой форматтер (prettier / ruff / gofmt) — см. комментарий-инструкцию внутри файла.
