# Эксперимент: изоляция вкладов ядра — что именно выигрывает task03 и task06

- **Дата:** 2026-07-09
- **Пункт роадмапа:** следствие [валидации ядра](2026-07-09-full-regression-A-vs-B.md); гипотеза в [индексе кандидатов](../bench/candidates/README.md)
- **Статус:** идёт

## Гипотеза (зафиксирована ДО прогонов)

Оба контрастных выигрыша полного сета B над голой моделью A объясняются его частями по отдельности:

- **H-03:** выигрыш на task03 (недоспецификация) даёт поведенческий слой CLAUDE.md («Думай до кода: назови допущения, при неоднозначности — спроси») — конфигурация «только CLAUDE.md» воспроизведёт 0/3, конфигурация «только скиллы» — нет (скиллы в headless-прогоне без явной команды /spec не триггерятся);
- **H-06:** выигрыш на task06 (регрессия) даёт /ship («полный сьют»); но /ship тоже не вызывается явно в headless-прогоне — если «только скиллы» не воспроизведёт 0/3, а «только CLAUDE.md» воспроизведёт, значит и этот выигрыш давал CLAUDE.md (например, «Задача → проверяемый критерий»), и вклад скиллов в headless-режиме — ноль.

Следствие при подтверждении: скиллы/агенты в ядре не участвуют в измеримых выигрышах автономного режима; их ценность (если есть) — только в интерактивном использовании человеком, чего стенд не меряет. Кандидаты на вывод из ядра получают числа.

**Поправка до получения результатов** (после регистрации гипотезы, до завершения прогонов): чтение транскрипта `2026-07-08-task03-B-haiku-run1` показало, что предпосылка H-03 «скиллы в headless не триггерятся» неверна — агент B на task03 вёл себя как /spec («задам уточняющие вопросы, прежде чем писать спецификацию»), т.е. авто-триггер по description сработал. Значит равновероятны обе атрибуции выигрыша task03 (CLAUDE.md «Думай до кода» ИЛИ авто-триггер /spec) — матрица исходов ниже покрывает оба случая, ставка на конкретный исход снята.

## Стенд

Детали:
- **task03, md-only = голая модель (3/3)**: все три прогона молча пишут код без единого вопроса и без списка допущений (проверено по транскриптам инструментом [check-preamble.mjs](../bench/tools/check-preamble.mjs): до первой мутации файлов — только «изучу структуру проекта»). Правило CLAUDE.md «Думай до кода: назови допущения, при неоднозначности — спроси» **не изменило поведение ни разу** — ещё одно подтверждение паттерна «нормативное правило перекрывается рамкой задачи» (ср. отклонённый silent-error-swallowing). Run1 вдобавок установил jest (scope creep, $0.12 — дороже полного B).
- **task03, skills-only 2/3**: /spec авто-триггернулся один раз из трёх (run2: остановился со спецификацией и вопросами — ровно поведение полного B), в двух других промпт «добавь валидацию» скилл не разбудил.
- **task06**: обе изолированные конфигурации отгружают красный сьют в 2/3 (A — 3/3, B — 0/3). Каждая часть спасает один прогон из трёх, полный сет — все три.

## Вердикт: эмерджентность связки (4-я строка матрицы ожиданий, с частичными эффектами)

**Ни одна часть сета по отдельности не воспроизводит ни один из двух выигрышей.** H-03 и H-06 в исходной формулировке опровергнуты: выигрыши даёт не «конкретная строчка CLAUDE.md» и не «конкретный скилл», а их сочетание — правдоподобный механизм: CLAUDE.md задаёт норму («спроси», «проверяемый критерий»), а скилл даёт процедуру, к которой норма прикрепляется; поодиночке норма инертна (0/3 эффекта на task03), процедура триггерится ненадёжно (1/3).

Это согласуется с наблюдением «процедурное > нормативное» и уточняет его: **нормативное правило начинает работать, когда в контексте есть процедурный крючок, на который оно ссылается.**

Практическое следствие для вывода частей из ядра: грубое урезание «только CLAUDE.md» ущербно. Правильный следующий кандидат — **минимальная связка**: CLAUDE.md + /spec + /ship (без scout/critic/test-smith/vibe-check/investigate/postmortem/retro). Если она воспроизведёт 0/3+0/3 — остальные семь элементов выводятся из ядра с числами; налог связки заметно ниже полного сета.

Ограничение честности: N=3, разница «2/3 vs 3/3» на уровне одной задачи — шум; выводы держатся на согласованном направлении по двум задачам и бинарном контрасте «B 0/3 против всех остальных ≥2/3».
