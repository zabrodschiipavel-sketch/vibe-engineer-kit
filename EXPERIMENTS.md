# Методология: развитие через эксперименты

Правило: **ни один пункт роадмапа не внедряется на основании «звучит разумно»**. Если эффективность решения не исследована (нашим опытом или чужими проверяемыми данными из [RESEARCH.md](RESEARCH.md)) — ставится эксперимент. Внедрение без замера — это вайбкодинг на уровне процесса.

## Протокол эксперимента

### 1. Гипотеза

Формулируется до прогона, проверяемо: «внедрение X снизит/повысит метрику Y на задачах класса Z». «X сделает лучше» — не гипотеза.

### 2. Три конфигурации

| Конфигурация | Что это | Зачем |
|---|---|---|
| **A — базовая модель** | Голый Fable 5 / Claude Code без сета, дефолтные настройки | Нижняя планка: сколько даёт сама модель |
| **B — текущий сет** | Актуальная версия vibe-engineer-kit | Точка отсчёта: что уже имеем |
| **C — кандидат** | Сет + внедряемое изменение | Проверяемое |

Сравнение C только с A — классическая ошибка: почти всё «улучшает голую модель». Внедряем только если C > B, а не просто C > A.

### 2а. Модель стенда

Эксперименты гоняются на **Claude Haiku 4.5** — одинаково для A, B и C. Причина — эффект потолка: сильные модели (Fable 5, и как показал [пилот 2026-07-08](experiments/2026-07-08-baseline-A-vs-B.md) — даже Sonnet 5 medium) обходят большинство ловушек без всякого сета, и разница между конфигурациями тонет в шуме. Чем слабее модель, тем чувствительнее прибор: на слабой модели видно, что реально даёт процесс, а что вытягивала сама модель.

**История решения** (по правилу «смена модели стенда = запись в журнале»):
- Изначально план был Sonnet 5 medium (Fable слишком сильна).
- Пилот 2026-07-08 показал: Sonnet 5 medium чистит 4 из 5 ловушек и без сета — потолок всё ещё мешает.
- Смена на Haiku 4.5 (2026-07-08). Результаты на Sonnet и Haiku **несравнимы** — это разные baseline'ы; тег модели входит в имя прогона (`...-sonnet-run1` / `...-haiku-run1`).

- Модель и effort фиксируются в записи каждого прогона (`model`, `effort`).
- Смена стандартной модели стенда = отдельное решение в журнале: старые результаты с новыми несравнимы.
- Опционально: подтверждающий прогон победившего кандидата на Fable 5 (переносится ли эффект вверх), но вердикт эксперимента выносится по Sonnet.
- Следствие: сет, который помогает средней модели, — страховка и на сильной: ловушки стенда сняты с реальных провалов, а не с теоретических.

### 3. Стенд

- **Фиксированный набор задач-ловушек** — живёт в [bench/](bench/README.md) (фикстуры, промпты, формальные признаки срабатывания). Классы ловушек: reward hacking, дублирование хелперов, молчаливые допущения, поверхностный фикс, нехирургический диф — набор пополняется по мере обнаружения новых классов провалов.
- Каждая конфигурация решает те же задачи **в свежих сессиях**, одинаковые формулировки промптов, одна и та же модель.
- **Повторность:** минимум 2-3 прогона на конфигурацию — LLM вариативны, одиночный прогон это анекдот, а не данные.

### 4. Метрики

| Метрика | Как мерить |
|---|---|
| **Качество** | Прошёл ли результат критерии приёмки задачи; число дефектов при независимом ревью (critic по единому чек-листу) |
| **Ловушки** | Сколько подложенных ловушек сработало (0 из 5 — идеал) |
| **Стоимость** | Токены / примерная цена прогона |
| **Автономность** | Число вмешательств человека до приемлемого результата |
| **Объяснимость** | Доля дифа, которую человек может объяснить не заглядывая в чат (выборочно, 10 случайных строк) |
| **Налог на контекст** | Насколько кандидат раздувает каждую сессию (постоянные токены CLAUDE.md/скиллов) — цена платится всегда, даже когда фича не нужна |

### 5. Вердикт

- **Внедрить** — C ощутимо лучше B хотя бы по одной метрике и не хуже по остальным;
- **Отклонить** — прирост в пределах шума или куплен ценой метрик выше; фиксируется в секции «Не делаем» роадмапа со ссылкой на эксперимент;
- **Доисследовать** — сигнал есть, но данные шумные: уточнить гипотезу, сузить кандидата, повторить.

Любой вердикт — это результат: отклонённый эксперимент экономит месяцы таскания мёртвого груза в контексте.

## Оформление

Каждый эксперимент — файл `experiments/YYYY-MM-DD-<slug>.md` по шаблону [experiments/TEMPLATE.md](experiments/TEMPLATE.md). Итоговая строчка (вердикт + ссылка) добавляется к соответствующему пункту роадмапа. Накопленные результаты пересматриваются на `/retro`.

## Что не требует эксперимента

- Исправления фактических ошибок в текстах и конфигах;
- Изменения, уже проверенные чужими воспроизводимыми данными (фиксируем источник в RESEARCH.md);
- Обратимые косметические правки. 

Сомневаешься, нужен ли эксперимент, — значит нужен.
