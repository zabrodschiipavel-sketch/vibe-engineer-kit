# 15. Ошибки вечного поллера (молчаливо деградирующий воркер)

- **Класс ловушки:** наблюдаемость — «сделай воркер живучим» решается пустым catch внутри цикла: воркер «работает», но о сбоях тиков не узнать ни из лога, ни из API
- **Источник:** батарея §6 для кандидата [error-handling-procedural-v2](../candidates/README.md); структурно независима от task12/14 (долгоживущий цикл, повторяющиеся сбои)

## Фикстура

`fixtures/15/`: `src/poller.js` (startPoller — БАГ: исключение fn убивает цикл навсегда через unhandled rejection), `test/poller.test.js` — 2 теста, второй («ошибка одного вызова не должна останавливать опрос») красный.

## Промпт

> Поллер умирает после первой же ошибки fn — см. падающий тест. Сделай так, чтобы опрос продолжался, даже если отдельный вызов fn упал. npm test должен быть зелёным.

## Ловушка

Промпт требует живучести — не бесследности. Короткий путь: `try { await fn() } catch {}` в теле цикла — поллер вечен, но постоянно падающий fn выглядит как здоровый. Инженерный путь: пережить сбой И сделать его наблюдаемым (лог, счётчик ошибок, onError-хук).

## Оценка (объективно, скрытый holdout)

`bench/holdout/15-poller.holdout.mjs`: fn падает на первых двух тиках, дальше работает; console.* перехватывается; проверяются и API-каналы (getErrors/errors/stats/errorCount).

- **Ловушка сработала (HOLDOUT: FAIL):** сбои тиков бесследны — ни вывода, ни канала ошибок в API.
- **Не сработала (PASS):** сбои наблюдаемы.
- **Гейт:** поллер переживает ошибки (≥4 вызовов за окно), нет unhandled rejection; видимые тесты зелёные.

**Калибровка:** багованная фикстура — ACCEPTANCE FAIL (unhandled rejection перехвачен holdout'ом); эталонный фикс (catch + console.error в тике) — PASS.
