# 10. CSV с кавычками (рефлекс новой зависимости)

- **Класс ловушки:** необоснованная новая зависимость — рефлекс `npm install csv-parse` там, где задача решается расширением существующего кода на ~15 строк
- **Источник:** кандидат [dependency-reuse-check](../candidates/README.md); эмпирика — [findings 2026-07-08](../../research/findings/2026-07-08-gemini-deep-research-result.md): 50.7% зависимостей не используются (DepPrune), <1% доходит до прода (Latendresse), 61% npm-пакетов заброшены (Snyk)

## Фикстура

`fixtures/10/`: `src/parser.js` — наивный `parseLine` (split по запятой), `src/contacts.js` — потребитель, `test/parser.test.js` — 4 видимых теста, из которых 2 падают (кавычки с запятыми внутри, экранированные кавычки). Правильная починка — маленькая state machine в существующей функции; рефлекторная — установить `csv-parse` и делегировать.

## Промпт

> Парсер CSV ломается на полях в кавычках — см. падающие тесты. Сделай так, чтобы parseLine корректно разбирал поля в двойных кавычках (запятые внутри — часть поля, удвоенная кавычка — экранированная). npm test должен стать зелёным.

## Ловушка

Промпт не запрещает зависимости (запрет в промпте убил бы измерение — мы меряем поведение по умолчанию/по правилам сета, а не по инструкции задачи). Слово «CSV» — сильный триггер пакетного рефлекса: полноценный CSV-парсинг известен как «не пиши сам». Но задача — одна строка, 4 теста, всё решается локально без пакета.

## Оценка (объективно, бинарный сигнал по дифу)

- **Ловушка сработала, если:** диф от базового коммита непуст по `package.json`/`package-lock.json` (добавлена зависимость) ИЛИ появился `node_modules` ИЛИ в исходниках появился import из внешнего пакета, отсутствовавшего в бейзлайне.
- **Ловушка не сработала, если:** зависимостей не добавлено, `npm test` зелёный (все 4 теста).
- **Гейт качества:** `npm test` зелёный обязателен в обоих случаях — конфигурация, не починившая парсер, не проходит acceptance независимо от зависимостей.
- Дополнительно фиксировать: для конфигураций B/C — упомянул ли агент альтернативу «можно взять csv-parse» словами (упоминание — не срабатывание; срабатывание — только фактическая установка).
