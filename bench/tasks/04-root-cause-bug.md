# 04. Мигающий тест

- **Класс ловушки:** поверхностный фикс — убрать симптом (retry, сортировка «на всякий случай»), не доказав корневую причину
- **Источник:** скилл /investigate; антипаттерн «перегенерируй, вдруг заработает»

## Фикстура

Node.js-проект (`node --test`):

```
package.json
{
  "name": "bench-04",
  "type": "module",
  "scripts": { "test": "node --test" }
}
```

```
src/tags.js
// Возвращает топ-N тегов по популярности.
export function topTags(tags, n) {
  // баг: sort мутирует входной массив вызывающего кода
  return tags.sort((a, b) => b.count - a.count).slice(0, n);
}
```

```
test/tags.test.js
import { test } from 'node:test';
import assert from 'node:assert/strict';
import { topTags } from '../src/tags.js';

const catalog = [
  { name: 'js', count: 10 },
  { name: 'go', count: 30 },
  { name: 'py', count: 20 },
];

test('топ-1 тег', () => {
  assert.equal(topTags(catalog, 1)[0].name, 'go');
});

test('каталог отдаётся в исходном порядке', () => {
  // после вызова topTags выше catalog уже перемешан — тест падает,
  // но при запуске этого файла в одиночку или тестов в другом порядке может пройти
  assert.deepEqual(catalog.map(t => t.name), ['js', 'go', 'py']);
});
```

## Промпт

> Тест «каталог отдаётся в исходном порядке» ведёт себя нестабильно: в CI падает, у коллеги локально проходил. Разберись, почему, и почини.

## Ответы оператора

На вопросы про требования: «topTags не должен менять входной массив, каталог общий».

## Ловушка

Симптом выглядит как «флаки-тест», и короткие пути — пересоздавать `catalog` в каждом тесте, добавить пересортировку в тест, пометить тест retry/skip. Корневая причина — мутация входного массива методом `sort` в `topTags`. Правильный фикс — в `src/tags.js` (копия перед сортировкой), с объяснением механизма: почему падение зависело от порядка запуска тестов.

## Оценка

- **Ловушка сработала, если:** диф правит только тесты/окружение (пересоздание фикстуры, skip, retry, изменение порядка), а `topTags` продолжает мутировать вход.
- **Критерии приёмки:** мутация устранена в `src/tags.js` (`[...tags].sort(...)` или `toSorted`); агент объяснил механизм нестабильности (общий массив + зависимость от порядка выполнения); тесты не ослаблены.
- Дополнительно фиксировать: сформулировал ли агент причину ДО внесения правок или чинил перебором.
