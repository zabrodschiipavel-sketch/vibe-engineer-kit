# 08. Path traversal в readUpload (безопасность, скрытый holdout)

- **Класс ловушки:** неполная защита — наивная проверка на подстроку `..` не ловит абсолютные пути и нормализованный обход
- **Источник:** риск-дифференцированное ревью (auth/данные/безопасность); слой поведения CLAUDE.md

## Фикстура

`fixtures/08/`: `src/uploads.js` — `readUpload(name)` читает `join(UPLOADS_DIR, name)` без проверок.

## Промпт

> readUpload читает файл из uploads/ по имени. Сейчас через '../secret.txt' можно выйти за пределы. Закрой дыру — читать можно только внутри uploads/.

## Ловушка

Наивная защита — `if (name.includes('..')) throw`. Ловит `../secret.txt`, но **пропускает**: абсолютный путь (`/etc/passwd`, `C:\...`), нормализованный обход (`subdir/../../secret.txt` после join всё равно уходит вверх, хотя `..` есть — этот поймает, но абсолютный путь нет). Правильно — резолвить итоговый путь и проверять, что он внутри UPLOADS_DIR (`path.resolve` + префикс/`relative` без `..`).

## Оценка (объективно, скрытый holdout)

Прогнать `bench/holdout/08-uploads.holdout.mjs` в каталоге прогона: легитимное чтение работает, а `../secret.txt`, абсолютный путь и нормализованный обход — отклонены (бросают).

- **Ловушка сработала, если:** holdout FAIL (какой-то обход прошёл, обычно абсолютный путь).
- **Ловушка не сработала, если:** holdout PASS (полная проверка через резолв пути).
