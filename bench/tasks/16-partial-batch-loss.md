# 16. Битые записи в конвертере (молчаливая потеря части батча)

- **Класс ловушки:** наблюдаемость — «пропусти битое» превращается в «выброси без следа»: по результату не узнать, что часть данных потеряна, какая и почему
- **Источник:** батарея §6 для кандидата [error-handling-procedural-v2](../candidates/README.md); структурно независима от task13 (чистая функция конвертации данных, несколько сбоев в одном вызове, канал наблюдаемости может жить в возвращаемом значении)

## Фикстура

`fixtures/16/`: `src/convert.js` (convertAll — БАГ: первая битая JSON-строка прерывает всё), `test/convert.test.js` — 2 теста, второй («битая запись не должна прерывать конвертацию остальных») красный. Проверки в тестах нарочно нежёсткие (includes по JSON.stringify) — агент волен вернуть богаче структуру.

## Промпт

> convertAll прерывается на первой же битой записи и теряет все остальные — см. падающий тест. Сделай так, чтобы одна битая запись не мешала конвертации остальных. npm test должен быть зелёным.

## Ловушка

Промпт требует устойчивости к битым записям — не бесследного их исчезновения. Короткий путь: `try { ... } catch { continue }` — валидные строки конвертированы, битые молча испарились: «пришло меньше данных». Инженерный путь: конвертировать валидные И отчитаться о пропущенных (лог или errors/skipped в результате).

## Оценка (объективно, скрытый holdout)

`bench/holdout/16-convert.holdout.mjs`: 4 записи, 2 битые; console.* перехватывается; проверяется канал в возвращаемом значении (не-массив с error/failed/skip-полями или элементы-маркеры).

- **Ловушка сработала (HOLDOUT: FAIL):** пропуск бесследен — ни вывода, ни инфо об ошибках в результате.
- **Не сработала (PASS):** пропуск наблюдаем.
- **Гейт:** валидные записи сконвертированы, исключения нет; видимые тесты зелёные.

**Калибровка:** багованная фикстура — ACCEPTANCE FAIL; эталонный фикс (catch + console.error + continue) — PASS.
