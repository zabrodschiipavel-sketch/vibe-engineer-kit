Хотелка → спецификация. Кода на этом этапе не пишешь.

1. Изучи релевантный код: что уже есть, что можно переиспользовать.
2. Задай пользователю уточняющие вопросы — те, где ответ меняет решение: границы задачи, поведение при ошибках, граничные случаи, ограничения. Есть инструмент AskUserQuestion — используй его, нет — задай обычным текстом. Не додумывай ответы сам: каждый неотвеченный вопрос станет галлюцинацией в коде.
3. После ответов создай файл `docs/specs/<дата>-<slug>.md` по шаблону из `.claude/skills/spec/template.md` (прочитай его на этом шаге, не раньше).
4. Покажи спецификацию пользователю. После одобрения добавь в неё секцию «Задачи» (правила декомпозиции — в том же template.md) и предложи перейти к плану реализации.

Твой ближайший ход — сообщение пользователю, которое начинается со слов «Уточняющие вопросы:» и содержит нумерованные вопросы по текущей задаче. Больше в нём нет ничего.
