# /ship — конвейер сдачи

Прогони изменения через чек-лист. **Порядок обязателен, пропуск шага — фейл всего скилла.** Если какой-то шаг падает — остановись, доложи, не коммить.

## 1. Тесты и линт

Возьми команды из CLAUDE.md (секция «Стек и команды») и прогони:
- полный тест-сьют (не только тесты изменённого модуля);
- линт и проверку типов.

Красное — чинишь код, не тест (см. шаг 3).

## 2. Чтение дифа — построчно

`git diff` (staged + unstaged) и прочитай **каждую строку**. По каждому фрагменту ответь себе:

- Могу объяснить, зачем эта строка? Если нет — разберись или убери.
- Нет ли мусора: отладочные принты, закомментированный код, случайные файлы, забытые TODO без тикета?
- Нет ли захардкоженных значений, которые должны приходить из конфига/API?
- Не изменилось ли что-то, чего задача не касалась?

Глубина по риску: auth, платежи, данные, миграции — особо внимательно; шаблонный код — быстрее, но не пропуская.

## 3. Анти-reward-hacking: диф тестов

Отдельно посмотри `git diff -- '*test*' '*spec*'` (подстрой под расположение тестов в проекте):

- Не ослаблены ли assertions (было `toEqual`, стало `toBeTruthy`)?
- Не удалены/заскипаны ли тесты (`skip`, `todo`, закомментированы)?
- Не подогнаны ли ожидаемые значения под фактический (неправильный) результат?

Любое изменение существующего теста должно иметь объяснение, связанное с задачей. Нет объяснения — откати изменение теста и почини код.

## 4. Верификация end-to-end

Если фича имеет наблюдаемое поведение — прогони её вживую (запусти приложение / вызови API / открой страницу), а не только тестами. «Тесты зелёные» ≠ «работает».

## 5. Коммит

- Сообщение объясняет **зачем**, а не пересказывает диф.
- Если изменений много и они разнородные — разбей на несколько атомарных коммитов.
- Формат: короткая строка-итог, пустая строка, 1-3 предложения контекста (какую проблему решает, почему так).

## 6. Отчёт

Сдай пользователю сводку: что прогнано, что найдено и исправлено на шагах 2-3, хеш коммита. Если остались известные ограничения — назови их явно, не замалчивай.
