# Кандидаты и гипотезы

Индекс всех гипотез об улучшении сета. Планка внедрения в рабочий `project-template/CLAUDE.md` — [EXPERIMENTS.md §6](../../EXPERIMENTS.md): N≥10 независимых задач, полный регрессионный прогон по стенду, расчёт выгода/цена, проверка на обобщение и на дублирование с уже существующими принципами. Пока кандидат не прошёл планку — он живёт **только здесь**.

## Активные кандидаты

| Кандидат | Гипотеза (проверяемая форма) | Статус | Источник/обоснование (эмпирика: [findings 2026-07-08](../../research/findings/2026-07-08-gemini-deep-research-result.md)) |
|---|---|---|---|
| [dedup-grep-principle](dedup-grep-principle/) | После фикса бага, на который указал тест, grep по кодовой базе на структурно похожие места снижает долю непокрытых дублей бага, оставшихся непочиненными | 🔬⏳ пилот 3/3 на task09 ([эксперимент](../../experiments/2026-07-08-duplicate-bug-and-task08-n3.md)), нужна батарея N≥10 | **Частота подтверждена данными:** до 33% клонов несут перенесённый баг (Mondal et al., ICSME), 28.57% фиксов в клонах — именно размноженные ошибки, >50% багфиксов в mobile затрагивают клоны. Обнаружено task09: ни A, ни B не находят дубль без явного принципа |
| [silent-error-swallowing](silent-error-swallowing/) | Явный запрет на пустой `catch`/проигнорированный reject снижает долю скрытых ошибок, не всплывших в момент фикса | 📝 не пилотирован | **Частота есть, тяжесть НЕ доказана:** ~20% ошибок рантайма не попадают в логи (Harness), 70% организаций без тестов обработки исключений (Serebrenik et al.); но связка «пустой catch → реальный инцидент/CVE» в литературе отсутствует — половина формулы выгоды пустая |
| [dependency-reuse-check](dependency-reuse-check/) | Явная проверка «есть ли уже способ решить это без новой зависимости» снижает число сомнительных npm/pip install на тривиальные задачи | ❌ **отклонён** ([пилот 2026-07-08](../../experiments/2026-07-08-dependency-reflex-pilot.md)): голая Haiku 0/6 не ставит пакеты на двух классических приманках (CSV, UUID) — для агента ванильный код короче установки, класс провала в agentic-сессиях не проявляется; правило было бы мёртвым налогом | Индустриальная эмпирика сильная (50.7% зависимостей не используются — DepPrune; 61% npm заброшены — Snyk), но она про людей и большие проекты, не про агентов. Мета-урок: частота ошибки в индустрии ≠ частота у агента. Пересмотр — только для профилей больших проектов (v0.4) |
| [doc-sync](doc-sync/) | Явное требование обновлять doc-комментарий/пример при изменении поведения снижает долю устаревшей документации после фиксов | 📝 не пилотирован | **Частота высокая:** лишь 13-20% изменений кода синхронно обновляют комментарии (IEEE TSE 2024), 60% доков устаревает за 6 мес.; но влияние drift именно на галлюцинации агентов не измерено — механизм гипотезы правдоподобен, не доказан |

## Гипотезы, не оформленные кандидатами

| Гипотеза | Статус | Почему не сейчас |
|---|---|---|
| Следовать существующему стилю обработки ошибок при правке (не вводить второй параллельный паттерн) | ♻️ дублирует | Уже покрыто «Хирургические правки»: «match existing style, even if you'd do it differently». Если понадобится — заострить эту формулировку, не добавлять новую |
| Валидировать вход только на границах доверия (API/CLI/файл) | 💡 | Пересекается с плановым скиллом `/security-review` (ROADMAP v0.2) — рассмотреть там как часть аудита, не как строчку CLAUDE.md |
| Явная сверка формулы/расчёта с эталоном перед реализацией | 💡, низкий приоритет | task07 уже показал: и A, и B решают формульные баги верно без подсказки — сомнительно, что принцип что-то добавит |
| Осторожность с гонками/конкурентным доступом к общему состоянию | 💡, низкий приоритет | Дорого проектировать объективную фикстуру (нужна настоящая конкурентность); отложить |
| Осведомлённость о cost/N+1-паттернах при работе с БД | 💡 | Актуально для backend-профиля (ROADMAP v0.4 «Профили стеков»), не для универсального ядра — рассмотреть там |

## Как добавить новую гипотезу

1. Сформулируй проверяемо: «принцип X снизит долю провалов класса Y на задачах Z» (не «X — это хорошая практика»).
2. Проверь на дублирование с уже существующими пунктами CLAUDE.md (см. таблицу выше — не первый случай, когда идея оказывается уже покрыта).
3. Если не дублирует — скопируй `project-template` в `bench/candidates/<slug>/`, добавь ОДНУ формулировку с пометкой `[КАНДИДАТ, не пилотирован]`.
4. Добавь строку в таблицу «Активные кандидаты» этого файла.
5. Пилот — по протоколу EXPERIMENTS.md §1-5 (сначала нужна фикстура-ловушка в `bench/fixtures/`, лучше со скрытым holdout). Пилотный сигнал переводит статус в 🔬⏳, не в ✅.
6. Внедрение в `project-template` — только после прохождения §6 целиком.
