Каждый элемент, который платформа научилась делать сама, превращается из пользы в чистый налог на контекст (≈2000–2800 ток./сессию постоянного налога на весь сет, замер `bench/tools/context-tax.mjs`). ## Выводы

1. **Три элемента — первоочередные кандидаты на вывод из ядра:** `scout` (дублирует Explore почти полностью), `critic` (в ядре; в стенде остаётся как измерительный прибор), `/vibe-check` (двойное пересечение, эффект не показан). Совокупная экономия постоянного налога: ~300–400 ток./сессию (frontmatter) + меньше поверхности поддержки.
2. **Пункт v0.2 «/security-review» устарел** — платформа сделала это сама. Правильная работа: прогнать встроенный на task08 (path traversal) и решить, нужно ли что-то сверх.
3. **Правило гигиены на будущее** (добавить в процесс /retro): при каждом значимом релизе Claude Code — пробегать эту таблицу; элемент, который платформа догнала, выводится из ядра, как MCP «не пригодился за неделю — отключи».
4. Вывод элемента из ядра — изменение сета, по EXPERIMENTS.md требует как минимум регрессионной проверки, что B без элемента не хуже B с ним (дешёвая проверка: элементы из п.1 не участвуют ни в одном выигранном контрасте стенда).
