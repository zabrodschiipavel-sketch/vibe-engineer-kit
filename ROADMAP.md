# Roadmap

Принцип развития: сет растёт по собственной петле — каждая новая фича проходит через спеку и верификацию, приоритеты пересматриваются на `/retro`. Не добавляем 24-й скилл ради количества: каждый элемент должен убирать конкретный класс ошибок или экономить конкретное время. Источник идей — [RESEARCH.md](RESEARCH.md).

**Развитие через накопление исследований** (методология — [EXPERIMENTS.md](EXPERIMENTS.md)): если эффективность пункта не подтверждена исследованием или чужими воспроизводимыми данными — перед внедрением ставится эксперимент со сравнением трёх конфигураций: голая модель (A) → текущий сет (B) → сет с кандидатом (C). Внедряем только при C > B; «улучшает голую модель» аргументом не является. Результаты копятся в `experiments/`, вердикты приписываются к пунктам ниже.

**Планка внедрения в CLAUDE.md/скиллы ужесточена 2026-07-08** (EXPERIMENTS.md §6, урок из провала `dedup-grep-principle`): выигрыш на одной сконструированной фикстуре — не основание для правки общего файла. Постоянное внедрение требует N≥10 независимых задач, полного регрессионного прогона по всему стенду с числами и явного расчёта выгода/цена. Пилотный сигнал на одной задаче переводит кандидата в `bench/candidates/` для дальнейшей валидации, не в рабочий сет напрямую.

## v0.2 — Валидация ядра, безопасность и память

- [x] **Валидация ядра: полная регрессия A vs B** (приоритет №1 из критического разбора 2026-07-09) — **завершена 2026-07-09, сет остаётся**: [experiments/2026-07-09-full-regression-A-vs-B.md](experiments/2026-07-09-full-regression-A-vs-B.md). Матрица 13 задач × N=3: B контрастно выигрывает **2 класса** — недоспецификация (task03: A 3/3 → B 0/3) и регрессия (task06: A 3/3 → B 0/3); на остальных B = A при суммарной цене +25%; постоянный налог ≈2000–2800 ток./сессию ([context-tax.mjs](bench/tools/context-tax.mjs)). **Изолирующий эксперимент выполнен 2026-07-09** ([карточка](experiments/2026-07-09-core-isolation.md)): ни CLAUDE.md, ни скиллы по отдельности выигрыши НЕ воспроизводят (md-only 3/3+2/3, skills-only 2/3+2/3 ловушек против B 0/3+0/3) — эффект эмерджентный: норма из CLAUDE.md работает только при наличии процедурного крючка-скилла. **Пилот минимальной связки (CLAUDE.md + /spec + /ship) — опровергнут** ([карточка](experiments/2026-07-09-minimal-bundle-pilot.md)): task03 3/3 (как голая модель, /spec не проснулся ни разу), task06 1/3, дороже полного B. **Итог линии: вывод элементов из ядра «за ненадобностью» заблокирован данными** — полный сет пока единственная конфигурация с надёжными 0/3. **Зонд плотности** ([карточка](experiments/2026-07-09-density-probe-pilot.md)) дал дифференцированный ответ: выигрыш task06 воспроизводится вообще без тел скиллов (0/3 — работает процессный приор от описаний), выигрыш task03 требует настоящего тела /spec (контр-рефлексное «остановись и спроси»). Формула сета: описания = приор и триггеры, тела = контр-рефлексные процедуры.

## v0.3 — Автоматизация петли

- [ ] **Инсталлер** — `install.ps1` / `install.sh`: копирование шаблона в проект одной командой, выбор глобальной/проектной установки (паттерн: setup-скрипты gstack и superpowers).

- [~] **Evals** — стенд работает end-to-end ([bench/](bench/README.md): 9 задач + раннер (A/B/**C**-кандидаты) + метрики + автосбор + скрытые holdout'ы). Проведено 4 эксперимента, см. `experiments/`. **Ключевые находки:**
  - старые 5 ловушек слишком лёгкие (Sonnet и Haiku чистят 4/5 без сета);
  - **task06 (регрессия) — сет ловит то, что голая модель отгружает: 3/3 vs 3/3 красных** (дисциплина `/ship` «прогони все тесты»);
  - **task09 (дублированный баг без теста) — сет БЕЗ доработки не находит дубль (0/3), как и голая модель.** Гипотеза уточнена: `/ship` ловит регрессии (тест был и упал), но не непокрытые дубли (тест никогда не существовал) — для этого нужен проактивный grep;
  - **кандидат C с явным принципом «после фикса — grep на дубли» дал пилотный сигнал 3/3** на одной фикстуре — но это переобучение на одном тесте, не доказательство пользы (см. урок выше). Правило **откачено** из `project-template/CLAUDE.md`, живёт как задокументированный кандидат в `bench/candidates/dedup-grep-principle/`, ждёт валидации на ≥10 независимых задачах прежде чем снова рассматриваться на внедрение.
  - task08 (path traversal) закрыт при N=3 — прошлый вывод «B хуже A» был шумом одной генерации.
  **Дальше:** протокол автоответчика для интерактивных задач (task03-подобных); ещё ловушки класса «формальный сигнал есть/нет», чтобы обкатать разделение гипотезы дальше.
