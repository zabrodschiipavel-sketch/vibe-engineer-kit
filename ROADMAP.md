# Roadmap

Принцип развития: сет растёт по собственной петле — каждая новая фича проходит через спеку и верификацию, приоритеты пересматриваются на `/retro`. Не добавляем 24-й скилл ради количества: каждый элемент должен убирать конкретный класс ошибок или экономить конкретное время. Источник идей — [RESEARCH.md](RESEARCH.md).

## v0.2 — Безопасность и память

- [ ] **Скилл /security-review** — аудит агентного кода: секреты в коде/логах, инъекции, обход проверок доступа. Основа: [trailofbits/skills](https://github.com/trailofbits/skills), идеи `/cso` из gstack (OWASP + STRIDE).
- [ ] **Правила `.claude/rules/`** — готовые модульные файлы (security.md, testing.md, git-workflow.md) как опция к монолитному CLAUDE.md, с инструкцией когда переходить.
- [ ] **Память v2** — обкатать [claude-mem](https://github.com/thedotmack/claude-mem) (автокомпрессия сессий) на реальном проекте; по результатам — либо в сет, либо мотивированный отказ в журнале решений.
- [ ] **Хуки-библиотека** — 3-5 готовых хуков к включению: автоформат (prettier/ruff/gofmt), блок коммита при падающих тестах, напоминание про /spec при большом дифе без спеки.

## v0.3 — Автоматизация петли

- [ ] **Инсталлер** — `install.ps1` / `install.sh`: копирование шаблона в проект одной командой, выбор глобальной/проектной установки (паттерн: setup-скрипты gstack и superpowers).
- [ ] **Автотриггеры скиллов** — descriptions скиллов отшлифовать так, чтобы петля запускалась без явных слэш-команд (паттерн superpowers: «агент просто имеет суперсилы»).
- [ ] **Evals** — мини-набор проверок сета на себе: дать агенту типовые задачи с подложенными ловушками (reward hacking, хардкод, лишний рефакторинг) и мерить, ловит ли сет.
- [ ] **/vibe-check v2** — трекинг динамики: сравнение с прошлым прогоном (стало чище или грязнее), выгрузка отчёта в файл.

## v0.4 — Масштаб

- [ ] **Кросс-харнесс** — портируемость скиллов на Codex CLI / OpenCode / Cursor (методология отделена от инструмента; референс — таблица хостов gstack).
- [ ] **Токен-экономика для больших репо** — интеграция пре-индексации (codegraph / graphify / codemaps): scout получает карту кода вместо Grep-разведки с нуля.
- [ ] **Профили стеков** — надстройки над универсальным ядром: web (браузерная верификация, Playwright MCP), data/ML (HF MCP, эксперименты), backend (БД read-only MCP, миграции).
- [ ] **Workflow-шаблоны** — заготовки многоагентных прогонов для крупных задач: миграция, аудит, массовый рефакторинг.

## Не делаем (осознанно)

- **Свой MCP-сервер** — пока хватает готовых; писать свой = поддерживать свой.
- **Десятки агентов-персон** (путь SuperClaude/BMAD) — противоречит принципу минимума: три роли с чёткой задачей лучше двадцати с размытыми.
- **Свой форк системного промпта** — сет живёт поверх штатного Claude Code, не вместо него: меньше ломается при обновлениях.
