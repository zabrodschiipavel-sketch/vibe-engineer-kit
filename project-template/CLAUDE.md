## Архитектурные инварианты

<!-- Правила, которые нельзя нарушать. Примеры оставлены как образец формата — замени своими. -->

- Бизнес-логика не живёт в контроллерах/компонентах — только в сервисном слое.
- Все обращения к внешним API — через существующие клиенты в `src/clients/`, новых HTTP-вызовов «в лоб» не делаем.
- Схема БД меняется только через миграции.

## Конвенции

- Переиспользуй существующие хелперы и утилиты — перед созданием новой функции поищи похожую (Grep по репо).
- Новый код — со статической типизацией, без `any`/нетипизированных сигнатур.
- Тесты лежат рядом с кодом / в `tests/` — _выбери и зафиксируй_.

- Необратимые действия (удаление файлов/веток/данных, force push, reset --hard, правка миграций задним числом) — только после явного согласования с человеком.
